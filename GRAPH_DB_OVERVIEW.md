# 수능 영어 기출문항 그래프 DB — 프로젝트 개요

> 이 문서는 GRAPH_DB_GUIDE.md의 배경, 설계 근거, 핵심 결론을 요약한 문서입니다.

---

## 1. 프로젝트 배경

### 1.1 문제 인식
- 기존 수능 문항 생성 시스템은 **Lexile + 7가지 유형 + 3단계 난이도**로만 문항을 생성
- 종속절 비율, 어휘 분포, 응집성, 추상도, 글의 구조, 오답 설계 전략 등 **핵심 변수 6가지가 누락**
- 오답 설계가 LLM 자율에 전적으로 의존하여 품질 편차가 큼

### 1.2 왜 그래프 DB인가

수능 문항은 단순 텍스트가 아니라 **다층 관계의 결합체**이다.

```
하나의 수능 문항 = 지문 + 글의 구조 + 출제 의도 + 요구 능력 + 난이도 변수 + 오답 설계
```

| 작업 | RDB | 벡터 DB | 그래프 DB |
|------|:---:|:------:|:--------:|
| 조건 조합 검색 (주제+난이도+구조) | JOIN 지옥 | 부정확 | **정확** |
| 유사 구조 다른 주제 문항 탐색 | 어려움 | 불가능 | **관계 탐색** |
| 오답 패턴 기반 검색 | 가능 | 불가능 | **가능** |
| 난이도 변수 조합 → 정답률 예측 | 가능 | 불가능 | **관계+속성 결합** |

### 1.3 RAG+벡터DB vs 그래프DB 비교 결론

- **RAG+벡터DB**: "비슷한 느낌"의 지문 검색, 빠른 프로토타입에 적합
- **그래프DB**: 조건 조합 검색, 오답 설계, 출제 논리 재현에 압도적 우위
- **최적 전략**: RAG로 시작 → 점진적으로 그래프 DB 도입 (하이브리드)

## 2. 설계 구조 요약

### 2.1 노드 12종

| # | 노드 | 역할 | 분류 |
|:-:|------|------|:---:|
| 1 | **Question** | 문항 (학년도, 번호, 정답률, 변별도) | 핵심 |
| 2 | **Passage** | 지문 원문 + 텍스트 복잡도 변수 (종속절 비율, 수동태 비율 등) | 핵심 |
| 3 | **Topic** | 주제 (대/중/소분류, 출제 빈도) | 핵심 |
| 4 | **Structure** | 글의 논리 구조 (주장-근거-반론-결론 등 10종) | 핵심 |
| 5 | **QuestionType** | 문항 유형 (빈칸추론, 순서배열 등 7종) | 핵심 |
| 6 | **Skill** | 요구 인지 능력 (Bloom's Taxonomy 연계) | 핵심 |
| 7 | **VocabProfile** | 어휘 분포 (초/중/고급 비율, AWL, TTR) | 핵심 |
| 8 | **DistractorType** | 오답 유형 패턴 (부분일치, 반대논지 등 7종) | 핵심 |
| 9 | **CoherenceProfile** | 응집성 (Kintsch 3수준, 단절 위치, 추론 요구도) | 보조 |
| 10 | **AbstractnessMap** | 추상도 (문장별 점수, 변화 패턴, 유형별 비율) | 보조 |
| 11 | **DifficultyModel** | 난이도 예측 모델 (유형별 가중치, 성능 지표) | 보조 |
| 12 | **Vocabulary** | 개별 핵심 어휘 (수준, 빈도, 수능 출현 횟수) | 보조 |

### 2.2 엣지 17종

| 분류 | 엣지 | 설명 |
|:---:|------|------|
| 필수 (9) | HAS_PASSAGE, IS_TYPE, ABOUT_TOPIC, HAS_SUBTOPIC, HAS_STRUCTURE, REQUIRES_SKILL, HAS_DISTRACTOR, HAS_VOCAB_PROFILE, HAS_COHERENCE, HAS_ABSTRACTNESS | 문항의 기본 관계 |
| 권장 (5) | EFFECTIVE_FOR, COMMON_DISTRACTOR, PRIMARY_SKILL, SIMILAR_TO, HAS_DIFFICULTY_MODEL | 분석/패턴 관계 |
| 선택 (3) | CONTAINS_VOCAB, SYNONYM/ANTONYM/CONFUSABLE_WITH, EXPLOITS_GAP | 어휘/오답 심화 관계 |

### 2.3 핵심 변수 통합

#### 난이도 변수 (Passage 노드 속성)
- 지문 길이 (word_count, sentence_count)
- 문장 복잡도 (avg/max/min_sentence_length, variance)
- 구문 복잡도 (subordinate_clause_ratio, nested_clause_depth, passive_voice_ratio)
- Lexile 추정치

#### 어휘 분포 (VocabProfile 노드)
- 수준별 비율: elementary / intermediate / advanced
- 학술 어휘 비율 (AWL 기준)
- 문맥 단서 제공률
- 어휘 다양도 (TTR)

#### 응집성 — Standard of Coherence (CoherenceProfile 노드)
- Kintsch 모델 3수준: surface / textbase / situation_model
- 연결어 밀도, 지시어 중복률
- 응집성 단절 위치 및 심각도
- 추론 요구도, 배경지식 요구도

#### 추상도 (AbstractnessMap 노드)
- 전체 추상도 점수 (0~1)
- 문장별 추상도 흐름 배열
- 추상화 유형: conceptual / metaphorical / philosophical
- 구체적 장치 (예시 수, 비유 수, 일화 여부)

## 3. 데이터 입력 프로세스

```
[1] 기출문항 수집 (PDF → 텍스트, 정답률/변별도)
 ↓
[2] AI 자동 분석 (주제, 구조, 능력, 오답, 응집성, 추상도)
 ↓
[3] NLP 자동 측정 (단어수, 종속절, 어휘분포, 연결어 — Python/spaCy)
 ↓
[4] 전문가 검수 (구조, 오답, 응집성, 추상도 보정)
 ↓
[5] 그래프 DB 적재
 ↓
[6] 난이도 모델 학습/업데이트 (정답률 기반 회귀분석)
```

## 4. 난이도 예측 모델

### 4.1 미세조정 원리
- 기출 문항의 **실제 정답률**을 종속변수로 회귀분석
- **문항 유형별 독립 모델** (빈칸추론은 응집성 중심, 순서배열은 연결어 중심)
- 매년 신규 수능/모의고사 데이터로 가중치 업데이트

### 4.2 유형별 핵심 변수

| 유형 | 핵심 난이도 변수 | 주요 오답 패턴 |
|------|-------------|-------------|
| 빈칸추론 | 응집성 단절, 추상도 | 부분일치, 반대논지 |
| 순서배열 | 연결어, 지시어, 구조 | 표면적 연결 함정 |
| 요약완성 | 추상화 수준, 글 구조 | 과잉일반화, 부분일치 |
| 심경분위기 | 묘사 간접성, 감정 어휘 | 유사감정 혼동 |
| 주제요지 | 주제 명시도, 추상도 | 세부내용 오인, 확대해석 |
| 지칭추론 | 지칭 대상 수, 구문 거리 | 인접 명사 혼동 |
| 무관한문장 | 주제 표면 유사도 | 주제어 공유 함정 |

### 4.3 목표 성능

| 지표 | 1단계 | 최종 |
|------|:----:|:---:|
| MAE (평균절대오차) | ±8% | ±3% |
| R² (설명력) | 0.60 | 0.85 |

## 5. 구축 로드맵

| 단계 | 기간 | 작업 | 효과 |
|:---:|:---:|------|------|
| 1 | 2주 | 문항 + 유형 + 주제 + 정답률 | 기본 검색 |
| 2 | 2주 | 글의 구조 + 요구 능력 | 구조 기반 검색 |
| 3 | 3주 | 어휘 분포 + 텍스트 복잡도 (NLP) | 정량 분석 |
| 4 | 3주 | 응집성 + 추상도 (AI+전문가) | 정밀 분석 |
| 5 | 2주 | 오답 유형 분류 | 선지 설계 |
| 6 | 2주 | 난이도 모델 학습 | 난이도 예측 |
| 7 | 지속 | 신규 문항 추가 + 모델 업데이트 | 지속 개선 |

### 최소 데이터: 약 840문항 (수능+모의 최근 10년)

### 권장 기술 스택
- **그래프 DB**: Neo4j
- **NLP**: spaCy, NLTK
- **AI 분석**: Claude API
- **난이도 모델**: scikit-learn, XGBoost
- **시각화**: Neo4j Browser, Streamlit

## 6. 활용 시나리오

1. **조건 조합 검색**: "기후변화 + 빈칸추론 + 고난도 + 인과관계 구조" 문항 찾기
2. **유사 문항 탐색**: "이 문항과 구조는 같지만 주제가 다른 문항"
3. **오답 패턴 분석**: "빈칸추론에서 가장 효과적인 오답 유형"
4. **난이도 맞춤 생성**: "정답률 30% 수준 빈칸추론의 변수 조합 조회"
5. **출제 경향 분석**: "최근 5년간 응집성 수준 변화 추적"
